# Quiz App

A personal practice-problem app. Load question sets (modules) as JSON files,
then quiz yourself with LaTeX-rendered questions and an auto-grading results screen.

---

## Quick Start

### 1. Install dependencies (one time only)
Open a terminal in the project folder and run:
```
npm install
```
This installs esbuild (the bundler that compiles your TypeScript into browser JS).

### 2. Build and run
```
npm run dev
```
This starts esbuild in watch mode AND serves the app at:
```
http://localhost:8000
```
Open that URL in your browser. The app will reload automatically when you save a file.

### 3. Build only (no server)
```
npm run build
```
Then open `index.html` directly in your browser, or serve it with any static server.

## How to Add a New Module

A "module" is a set of questions. Each module lives in one JSON file.

### Step 1 — Create a new JSON file
Copy `modules/example-algebra.json` and rename it, e.g. `modules/my-calc.json`.

Fill it in using this format:
```json
{
  "id": "my-calc",
  "title": "Calculus I",
  "description": "Limits, derivatives, and basic integrals.",
  "image": "assets/calc.png",
  "questions": [
    {
      "id": 1,
      "prompt": "Find the derivative of $f(x) = x^2$.",
      "type": "text",
      "correctAnswer": "2x",
      "explanation": "Power rule: bring down the exponent and reduce it by 1."
    },
    {
      "id": 2,
      "prompt": "Which is the correct limit? $$\\lim_{x \\to 0} \\frac{\\sin x}{x}$$",
      "type": "multiple-choice",
      "choices": [
        { "label": "$0$",        "value": "0"    },
        { "label": "$1$",        "value": "1"    },
        { "label": "$\\infty$",  "value": "inf"  },
        { "label": "undefined",  "value": "undef"}
      ],
      "correctAnswer": "1"
    }
  ]
}
```

**LaTeX tips:**
- Wrap inline math in single dollar signs: `$x^2$`
- Wrap display (centered) math in double dollar signs: `$$x^2 + y^2 = r^2$$`
- In JSON strings, backslashes must be doubled: write `\\frac` not `\frac`

### Step 2 — Register the module in `modules/index.ts`
Open `modules/index.ts` and add two lines:
```typescript
// At the top, add an import:
import myCalc from "./my-calc.json";

// In the allModules array, add your variable:
export const allModules: Module[] = [
  exampleAlgebra as Module,
  exampleChemistry as Module,
  myCalc as Module,   // <-- add this
];
```

### Step 3 — Rebuild
Save the file. If `npm run dev` is running, it rebuilds automatically.
Refresh the browser and your new module appears on the home screen.

## Question Format Reference

| Field          | Required | Description |
|----------------|----------|-------------|
| `id`           | Yes      | A number, unique within the module. Used for tab labels. |
| `prompt`       | Yes      | The question text. Supports LaTeX with `$...$` and `$$...$$`. |
| `type`         | Yes      | Either `"text"` or `"multiple-choice"` |
| `correctAnswer`| Yes      | For text: the expected simplified answer (e.g. `"3"`). For multiple-choice: the `value` of the correct choice. |
| `choices`      | MC only  | Array of `{ label, value }` objects. `label` supports LaTeX. |
| `explanation`  | No       | Shown on the results screen if the user got it wrong. |

### How answer checking works
- **Text answers**: The user's input is run through math.js to simplify arithmetic.
  `"2+1"` becomes `"3"`, which matches `"3"`. Non-numeric expressions like `"6x"` are
  compared literally (lowercased, trimmed).
- **Multiple choice**: The selected `value` is compared to `correctAnswer` directly.

## Customization Tips

- **Change the color scheme**: Open `style.css` and edit the variables inside `:root` at the top.
- **Change the app title**: Edit the `<h1>` text in the `renderHome()` function in `src/app.ts`,
  and the `<title>` tag in `index.html`.
- **Add an image to a module**: Put a `.png` or `.jpg` file in the `assets/` folder,
  then set `"image": "assets/yourfile.png"` in the module JSON.
- **Adjust answer comparison**: The grading logic is in `renderResults()` in `src/app.ts`.
  Currently it lowercases and trims both sides before comparing.
